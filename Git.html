GIT + github.com

Работа с GIT + github. Подготовка

              
http://git-scm.com/downloads       # Загружаем GIT тут
$ ssh-keygen -t rsa -C "your_email@whatever.com" # Создание публичных ключей (Отправить владельцу репозитория для получения прав, либо самостоятельно загрузить в настройки профиля github,bitbucket, gitlab)
$ git config --global user.name "Your Name"   # имя пользователя (необходимая настройка)
$ git config --global user.email "your_email@whatever.com"   # e-mail пользователя (необходимая настройка)
 $ git init   # Создание репозитория (в папке проекта появится скрытая папка git) (Либо папку можно создать на github. Клонируем репозиторий на локальную машину: если ключи сгенерированы можно через SSH иначе через HTTPS)
$ git clone git@github.com:yurasts/git-demo.git   # клонирование проекта созданного на githab.com (где git@github.com:yurasts/git-demo.git SSH или HTTPS)

"ctrl"+"ins" - вставка в git bash  / "shift"+"ins" - копирование в git bash

Работа с папками в GIT BUSH

              
$ pwd               # текущий адрес
$ cd /c/data/te...  # переход в папку
$ cd ..             # выйти на 1 уровень выше
$ cd ../..          # выйти на 2 уровня выше
$ mkdir New_Proj    # создать каталог
$ cd !$             # перейти в только что созданную папку
$ cd name           # войти в папку
$ ls                # просмотр содержимого папки

$ git remote -v  # проверка действительно ли мы на github а не каком-то другом хостингеу

$ git status  # проверка на наличие изменений в рабочем каталоге 
 СТАТУС 1. index.html - новый файл не отслеживается системой git или в файл внесли изменения.
 СТАТУС 2. style.css - файл добавлен в отслеживание системой git.
 СТАТУС 3. После commita, т.е. создания контрольной точки, добавленных в отслеживание файлов при выполнении команды "git status" не будет видно 

            
              
Добавление файлов в состояние "отслеживание" 
$ git add -A          # все файлы
$ git add .           # все файлы
$ git add style.css   # один файл
$ git add *.css       # группу файлов

$ git reset           # отмена добавления в отслеживание

            
              
Добавление файлов в состояние Commit и передача на github 
$ git commit -m "add file song.txt"  # добавление комментария об изменении файла
$ git push   # добавление изменений на github
$ git pull   # подтягивание новых данных с github
$ git log    # просмотр истории коммитов

Изменение или отмена действий

Working directory

              
$ git checkout -- file.txt - откат изменения в одном конкретном файле
$ git checkout . - возврат всех файлов в исходное состояние которое известно git

ВЕРНУТЬ КОНТЕНТ КОТОРЫЙ БЫЛ ДО "checkout" НЕВОЗМОЖНО

              
$ git clean -xdf - удаляет новые файлы

флаги: x - игнорирует файл ignor, d - удаляет в т.ч. и дирректории, f - force без неё ничего не произойдет

Staging area (index)

              
$ git reset -- file.txt - возвращает файл из коммита в файловую систему (git gui& клик по иконке в нижней части переместит в верхнюю)
$ git reset --hard - возвращает файл из коммита в файловую систему с удалением всех изменений

Local branch (КАК ОТМЕНИТЬ КОММИТ)

              
$ git reset --soft HEAD~1                   - отменить 1 коммит, контент изменений сохранится, файл в индексе
$ git reset --mixed HEAD~1                  - отменить 1 коммит, контент изменений сохранится, файл НЕ в индексе
$ git reset --hard HEAD~1                   - отменить 1 коммит, контент изменений НЕ сохранится
$ git commit --amend -m "commit message"    - в случае если в коммит нужно что-то дописать

Работа с ветками

создание, пререход, объединение веток

              
$ git checkout -b feature    - cоздание ветки с имененм feature
$ git checkout main          - перейти в ветку main
$ git merge feature          - объединение ветки "main" с "feature", окончательный комит в gitgui&

tags

              
$ git tag ver1        - маркировка коммита тегом ver1
$ git tag --list      - view tags
$ git push --tags     - push tag
$ git checkout ver1   - Check it out

rebase

              
$ git rebase main - родительская точка главной ветки (main) перемещается на актуальную

remote

              
$ git remote add name url    - новоe подключения к удаленному репозиторию.Можно отредактировть файл /.git/config
$ git remote rm name         - удаление подключения к удаленному репозиторию с именем
$ git remote add origin git@github.com:user/repo.git     - подключение к новому репозиторию
$ git push --set-upstream origin main     - git push автоматически создаст ветку в удаленном репо, если это необходимо
$ git remote –v              - просмотреть к какому репозиторию относимся
$ git remote show            - посмотреть имя
