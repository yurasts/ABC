GIT + github.com

Загрузить гит можно тут:

              
http://git-scm.com/downloads

Создание публичных ключей (id_rsa)

              
$ ssh-keygen -t rsa -C "your_email@whatever.com"

Отправить владельцу репозитория для получения прав работы, либо самостоятельно загрузить в настройки профиля в bitbucket, github, gitlab

Настройка имени пользователя и e-mail:

              
$ git config --global user.name "Your Name"
$ git config --global user.email "your_email@whatever.com"

Создание репозитория

В каталоге проекта используя командную строку

              
$ git init

В папке проекта появится скрытая папка git

Либо в github. Клонируем репозиторий на локальную машину: если ключи сгенерированы можно через SSH (иначе через HTTPS), копируем

переходим в папку где будет git сопровождение

              
$ cd /c/data/temp

проверка папки назначения:

              
$ pwd

клонируем проект созданный на githab.com (SSH или HTTPS)

              
$ git clone git@github.com:yurasts/git-demo.git

ctrl+ins - вставка в git bash / shift+ins - копирование в git bash

list убедимся что создалась директория

              
$ ls

переходим в папку

              
$ cd /c/data/temp/git-demo

проверка действительно ли мы на github а не каком-то другом хостингеу

              
$ git remote -v

проверка на наличие изменений в рабочем каталоге:

              
$ git status

index.html - новый файл не отслеживается системой git или в файл внесли изменения (статус 1)

style.css - файл добавлен в отслеживание системой git (статус 2)

После commita, т.е. создания контрольной точки, добавленных в отслеживание файлов при выполнении команды "git status" не будет видно (статус 3)

Добавление файлов в состояние "отслеживание" или индексировать изменения

Можно добавлять все файлы, один конкретный файл или группу файлов

              
$ git add -A
$ git add style.css
$ git add *.css

git теперь знает об изменении, но изменение пока не записано в репозиторий.

Если вы решили, что не хотите коммитить изменения, можно снять индексацию этих изменений:

Выполнить

              
git reset

Добавление файлов в состояние Commit и передача на githab

комментарий о добавлении измененного файла

              
$ git commit -m "add first 2 lines in the song.txt"

добавление изменений на github:

              
$ git push

обновление данных с github:

              
$ git pull

смотрим историю коммитов

              
$ git log

вспомогательные скины (упрощают работу)

              
$ git gui&
$ gitk

Изменение или отмена действий

Working directory

откат изменения в одном файле

              
$ git checkout -- file.txt

возврат всех файлов в исходное состояние которое известно git.

              
$ git checkout .

ВЕРНУТЬ КОНТЕНТ КОТОРЫЙ БЫЛ ДО "checkout" НЕВОЗМОЖНО

удаляет новые файлы

              
$ git clean -xdf

флаги: x - игнорирует файл ignor, d - удаляет в т.ч. и дирректории, f - сила без неё ничего не произойдет

Staging area (index)

возвращае файл из статуса до "git add"

              
$ git reset -- file.txt

Local branch (КАК ОТМЕНИТЬ КОММИТ)

контент не нужен и содержимое можно удалить

              
$ git reset HEAD^^ (HEAD~2)

в случае если в коммит нужно что-то дописать

              
git commit --amend -m "commit message"

Merge (слияние)

абракодабра "enter"
ctrl+"C" два раза
:wq! "enter"
