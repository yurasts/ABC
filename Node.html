NodeJS

Создание нового среды:

              
$ mkdir mypackage/ 	# создание новой папки
$ cd mypackage/ 	# переход в папку
$ npm init      	# инициализирование проекта, создание файла package.json
$ npm init -y      	# ...с последующим редактированием в IDE
$ npm list -g      	# вывести все пакеты, которые установлены глобально

            
              
Добавление зависимостей разработки в проект:
$ npm install --save-dev fs-extra	# NPM
$ npm i -D fs-extra     	        # сокращенная версия
$ yarn add --dev fs-extra            	# Yarn
$ yarn add -D fs-extra            	# сокращенная версия

            
              
Глобальная установка зависимостей:
$ npm install --global fs-extra	# NPM
$ npm i -g fs-extra          	# сокращенная версия
$ yarn global add json      	# Yarn
$ npm search hook.io      	# Поиск пакетов в npm

            
              
Проверка и обновление зависимостей:
$ npm outdated	              # просмотр прямых устаревших зависимостей
$ npm outdated --depth=9999   # просмотр всех устаревших зависимостей
$ npm update	              # обновление прямых устаревших зависимостей с учетом semver
$ npm update --depth=9999     # обновление всех устаревших зависимостей с учетом semver
$ npm-check 	              # просмотр прямых устаревших зависимостей
$ npm-check -u	              # интерактивное обновление прямых устаревших зависимостей

Creating a NodeJS project

              
creating a folder for the project (ex.:Unit Testing Project II) - создаём папку для проекта
Open folder (m.b.: c:\Data\Unit Testing Project II\)  - открываем папку в Visual Studio Code 
PS C:\Data\Unit Testing Project II>npm init  - инициализируем проект (находясь в папке проекта) install a package and save it as a dependency in the package.json file
package name: (unit-testing-project-ii) - имя (в скобках по умолчанию - enter)
version: (1.0.0) - enter
description: a simple unit testing project using mocha - описание
entry point: (index.js) - точка вхождения (по умолчанию, можно изменить если понадобится)
test command: mocha - test command
git repository: - устанавливается автоматом если есть (если не используется по умолчанию - enter)
keywords: mocha unit testing - не обязательное поле
author: - по умолчанию - enter
license: (ISC) - в скобках по умолчанию - enter
About to write to C:\Data\Unit Testing Project II\package.json:? - enter
Установка необходимых зависимостей
PS C:\Data\Unit Testing Project II> npm install eslint --save-dev - enter (установит в package.json секцию "devDependencies" и создаст папку "node_modules")
Is this OK? (yes) - enter
PS C:\Data\Unit Testing Project II> npm install mocha --save-dev - enter (по умолчанию скачивается последняя версия)
PS C:\Data\Unit Testing Project II> npx eslint --init - enter (инициализация линтера, формируется config, npx комманда для конфигурационной работы позволяет устанавливать не скачивая, а ставит пермонентно)
√ How would you like to use ESLint? · style - enter
√ What type of modules does your project use? · commonjs - enter (если версия Node 14 и выше можно использовать JavaScript modules (import/export))
√ Which framework does your project use? · none - enter
√ Does your project use TypeScript? · No / Yes - enter
√ Where does your code run? · browser, node - enter (в данном случае переключение с помощью пробела))
√ How would you like to define a style for your project? · guide - enter
√ Which style guide do you want to follow? · google  - enter
√ What format do you want your config file to be in? · JSON - enter
√ Would you like to install them now with npm? · No / Yes - enter
Is this OK? (yes) - enter
